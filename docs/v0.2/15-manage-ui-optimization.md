# v0.2 管理页 UI / 交互优化记录

> 目标：让高频操作围绕技能列表展开，减少筛选、问题修复和软连接管理之间的上下跳转。

## 本次调整

- 将 Agent 概览卡片改为同步控制台内的横向范围筛选条，保留包含 / 排除语义与 Agent 管理入口。
- 将搜索、排序、Agent 范围和问题修复合并到同一控制台，技能列表紧随其后。
- 将异常、状态和其他筛选默认收起到“更多筛选”，避免低频条件长期占用首屏高度；激活筛选时显示数量。
- 将问题修复改为控制台右侧的紧凑待处理分组，点击分组仍复用原有批量面板或单条定位逻辑。
- 保留列表 / 树视图、批量选择、批量同步、关系矩阵和 Agent 管理，不改变后端命令与数据模型。

## 交互规则

1. 默认进入页面后，搜索框、排序、Agent 范围和待处理入口位于同一可见区域。
2. 点击 Agent 胶囊进行包含筛选；再次点击顶部的包含 / 排除按钮切换筛选方向。
3. 低频状态条件通过“更多筛选”展开，多个条件继续采用 AND 逻辑。
4. 问题修复分组直接进入既有 `BatchSyncPanel`，不新增第二套修复模型。
5. 批量选择栏仍在技能列表底部浮动，适合连续处理多个技能。
6. 关系矩阵继续作为高级诊断区域，默认折叠。

## 验收重点

- 首屏不再依次堆叠 Agent 概览、状态筛选和问题修复三个大面板。
- 筛选结果计数、清除筛选和快捷键搜索行为保持可用。
- `src/components/manage/ManageTab.vue`、`IssueRepairPanel.vue` 与三份 locale 文件保持同步。
- 前端 `pnpm build` 与 Rust `cargo check` 无编译错误。
