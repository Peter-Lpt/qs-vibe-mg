# 功能待办与设计审计

这里只记录已经确认要继续推进的范围。目标不是把所有点都做完，而是先把最值得改、最能解锁后续能力的部分排清楚。

## 我先给出的初步判断

1. 先补 `Skill` 的安装与更新来源模型。
   - 这是后续支持 `npx`、git、Claude Marketplace / plugin、GitHub skill 仓库更新的底座。
   - 没有这层，后面的更新基本都会退化成一堆分支判断。

2. 再补来源识别与信任边界。
   - 哪些来源能明确识别，哪些只能 best-effort，要先说清楚。
   - 没有 provenance，就不要默认自动更新。

3. UI 壳和整体视觉要单独提到高优先级。
   - 现在的问题不是某一个按钮，而是整套头部、卡片、设置入口、版本号、更新入口、边框样式都不统一。
   - 这类问题会直接影响产品第一印象，应该早修。

4. `project_roots`、Agent 路径、同步安全这些能力要继续保留，但优先级排在来源模型和 UI 壳之后。

## 总体优先级

1. `Skill` 安装 / 更新 / 来源边界
   - 统一安装来源、更新路由、日志边界。
   - 覆盖 `npx`、git、Claude Marketplace / plugins、GitHub skill 仓库。
   - 明确哪些来源可识别，哪些只做 best-effort。
   - 先有来源元数据，再谈自动更新。

2. App 壳与整体视觉统一
   - 最小化 Win / Mac 适配差异，优先保证基础布局一致。
   - 重做左上角品牌区和顶部导航。
   - 统一按钮尺寸、间距、边框层级和卡片样式。
   - 版本号、检查更新、升级入口应放到设置中。

3. Agent 路径配置增强
   - `detect_dir` 可编辑。
   - 内置 agent 的 `skills_dir` 可覆盖。
   - 增加路径校验提示。

4. `AgentConfig` 能力模型完善
   - 增加 `additional_scan_dirs`。
   - 增加可配置的项目扫描模式。
   - 避免把所有项目约定永久写死在代码里。

5. 同步安全与 Windows 兜底
   - symlink / junction / copy 失败提示。
   - 权限说明与恢复路径。
   - 默认关闭破坏性项目操作。

6. 统一动作模型
   - 收敛 list / tree / detail / batch 的重复 action 逻辑。

7. 删除语义拆分
   - 区分 library skill、agent link、断链清理、真实副本删除。

8. 问题修复入口增强
   - 保留问题分组上下文。
   - 批量模式预设策略。

9. 基于 Git 的 canonical source 更新
   - 作为后续增强，不抢当前主线。

10. 文件 / 目录命名与布局
   - 作为命名层面的收尾项放后面。

## 需求分组

### A. Skill 安装 / 更新 / 来源边界

#### 必须加

- 记录安装来源。
- 记录安装命令或可回放的更新命令。
- 记录版本、commit、最后检查时间、信任等级。
- 更新时能路由回正确来源。
- 明确哪些来源可识别，哪些只能 best-effort。
- 先有来源元数据，再谈自动更新。

#### 可延后

- Marketplace / plugin 的深度解析。
- 从 README、`.git/config`、额外元数据里推断来源。

#### 不建议加

- 没有来源元数据就自动更新。
- 默认把所有日志全量暴露给界面。

#### 可延后

- 把来源识别做成很强的自动推断系统。
- 把所有来源统一抽象到非常复杂的资产体系里。

### C. App 壳与视觉统一

#### 必须加

- 左上角品牌区重做。
- 外层边框重复显示问题重构。
- 按钮尺寸、图标大小、间距统一。
- 设置入口重构。
- 版本号、更新检查、版本升级放入设置。

#### 可延后

- 单独拆成全新的设置页面。
- 进一步细化跨平台差异。

#### 不建议加

- 只改局部控件，不处理整体布局层级。

### D. 现有能力继续推进

#### 必须加

- Agent 路径配置增强。
- `AgentConfig` 扫描模型完善。
- 同步安全与 Windows 兜底。
- 统一动作模型。
- 删除语义拆分。
- 问题修复入口增强。

#### 可延后

- 基于 Git 的 canonical source 更新。
- 文件 / 目录命名与布局重构。

## 现有范围

1. Agent 路径配置增强
   - 可编辑 `detect_dir`。
   - 支持覆盖内置 agent 的 `skills_dir`。
   - 增加 agent 根目录与实际 skills 目录的路径校验提示。

2. `AgentConfig` 能力模型完善
   - 增加 `additional_scan_dirs`。
   - 增加可配置的项目扫描模式。
   - 避免把所有项目约定永久写死在代码里。

3. 同步安全与 Windows 兜底
   - 让 symlink、junction、copy 失败信息更清楚。
   - 说明权限要求和恢复路径。
   - 默认保持对项目源的破坏性操作关闭。

4. 统一动作模型
   - 减少 list、tree、detail、batch 各处重复的 action `switch` 逻辑。
   - 把公共动作行为收敛到同一套 action registry。

5. 删除语义拆分
   - 删除 library skill。
   - 移除 agent link。
   - 清理断链。
   - 删除 agent 的真实副本。
   - 默认保持 project 文件只读。

6. 问题修复入口增强
   - 进入批量模式时保留问题分组上下文。
   - 从选中的修复分组里预设合适的策略。
   - 避免只帮用户选中条目，却丢掉修复意图。

## 建议顺序

1. `Skill` 安装与更新来源模型。
2. 来源识别与信任边界。
3. App 壳与整体视觉统一。
4. Agent 路径配置增强。
5. `AgentConfig` 能力模型完善。
6. 同步安全与 Windows 兜底。
7. 统一动作模型。
8. 删除语义拆分。
9. 问题修复入口增强。
10. 基于 Git 的 canonical source 更新。
11. 文件 / 目录命名与布局。
